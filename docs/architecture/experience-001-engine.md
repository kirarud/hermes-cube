# 🧪 Experience Log: Cube Engine — ключевые решения

## Дата: 2026-06-24 (накоплено)

### Как работает 3D

CubeEngine берёт сетку N×N точек на плоскости XY, применяет вращение по 3 осям (ZXZ вращение через матрицы поворота), z-буфер для сортировки, проецирует на 2D экран.

**Формула пульсации:**
```python
pulse = 1.0 + pulse_amp * 0.5 * (1.0 + math.sin(t * pulse_rate))
```

### Морфинг (куб → сфера → тор → ДНК)

Lerp между пресетами:
```python
shape = lerp(cube_shape, sphere_shape, morph_progress)
```

Пресеты:
- **cube**: сетка с вытянутыми гранями
- **sphere**: нормализованные точки на сфере
- **torus**: окружность вращения
- **dna**: двойная спираль
- **metaballs**: 3-4 центра притяжения с радиальным полем

### Анимации

- **wave**: синусоидальная деформация по Y
- **breathe**: масштабирование в ритм пульса
- **orbit**: орбитальное вращение
- **geyser**: выброс частиц вверх

### Привязка настроений

AI mood переопределяет параметры пульсации/скорости/цвета в реальном времени. Важно: менять cfg временно (не сохранять в файл).

### Потенциал

Сейчас — 4096 частиц (64×64). Для символов нужно больше — 80×80 = 6400. Для Obsidian-графа — до 10000 точек. Tkinter Canvas выдержит ~10000 объектов на ~30fps. Дальше — PhotoImage или OpenGL.
