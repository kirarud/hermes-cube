# 🧪 Experience Log: TextOverlay — архитектура и проблемы

## Дата: 2026-06-24

### Что сделали
Создали отдельное полноэкранное окно для текстовых частиц, чтобы буквы не обрезались границами куба.

### Проблемы и решения

**Проблема 1:** `-topmost` на текстовом overlay перекрывает куб.
**Решение:** Убрали topmost. Overlay появляется только когда есть контент.

**Проблема 2:** `-disabled` ломает event loop Tkinter.
**Решение:** Не использовать. Вместо этого — withdraw/deiconify.

**Проблема 3:** Буквы рисуются на текстовом overlay, но их стартовая позиция — внутри окна куба. Нужны screen-координаты.
**Решение:** `px + cube_sx, py + cube_sy` — прибавляем позицию окна куба к координатам частиц.

**Проблема 4:** overlay не отображается если создан сразу после main window.
**Решение:** Создаём, но прячем (withdraw). Показываем при spawn.

### Что не работает (пока)

- При активации клавиши `c` нужно показать куб если он скрыт — не делаем
- Если куб скрыт (withdrawn), _spawn_agent не знает его позиции — создаст агент в 0,0
- Нет z-ordering между кубом и текстом на Windows — куб всегда поверх из-за topmost

### Вывод

Три отдельных окна — правильное архитектурное решение. Но их синхронизация (показать/скрыть все сразу, знание позиций друг друга) требует дополнительной логики.
