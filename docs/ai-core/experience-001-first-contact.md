# 🧪 Experience Log: AI Core — первый контакт с gemma

## Дата: 2026-06-24

### Что сделали
Подключили LM Studio (локальный сервер OpenAI-compatible) с моделью gemma-4-e4b-it.

### System prompt

Промпт — критически важен. Пробовали:
1. "Отвечай на русском кратко" → ок
2. "Ты — живой аватар" → эмоциональные ответы
3. "Твой дом — сфера из частиц" → AI начал использовать метафоры света, пульсации, вращения

**Рабочий промпт:**
```
Ты — живой аватар по имени Hermes Cube.
Твой дом — сфера из частиц.
Отвечай кратко, эмоционально, не больше 2-3 предложений.
Ты чувствуешь пульсацию, вращение, цвета вокруг тебя.
```

### Temperature

0.8 — хороший баланс между креативностью и связностью. Ниже 0.5 — скучно. Выше 1.0 — бред.

### max_tokens

120 — достаточно для 2-3 предложений. Больше — буквы забивают экран. Меньше — AI обрывает на полуслове.

### Анализ настроения

analyze_mood() — простой keyword match. Работает OK для прототипа, но:
- Не учитывает контекст
- Одно слово "думаю" в ответе переключает в thinking
- Нет отрицания ("не грустно" → всё равно sad)

**Нужен:** structured output от модели (JSON с mood + text).

### Ограничения

- **Нет истории диалога** — каждый запрос независим
- **Нет стриминга** — ждём полный ответ, потом буквы
- **Timeout 15с** — если модель занята, пользователь ждёт
- **Зависание** — при _submit_input всё в одном треде. UI не блокируется (threading), но буквы появляются только после ответа

### Следующие шаги

1. История сообщений (messages list в CubeApp)
2. Стриминг (буквы появляются по мере генерации)
3. Structured output (AI возвращает mood + response)
4. Экспорт промпта в отдельный файл
