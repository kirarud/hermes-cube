# 🧪 Experience Log: PixelGrid — прототип первой живой частицы

## Дата: 2026-06-24

### Что сделали
Реализовали первую версию агента-смайлика. По нажатию `a` из куба вылетает пиксельный спрайт.

### Технические детали

**Спрайты:** numpy matrix (uint8), где 0 = прозрачный, 1+ = индекс палитры.

**Палитра:** 7 цветов (прозрачный + 6 цветов куба).

**Canvas:** каждый пиксель спрайта = отдельный create_rectangle размером 3×3 пикселя. Да, это много объектов Canvas, но с 10×7 = 70 пикселей ~30-40 rect'ов работает ок. Для больших спрайтов нужен PhotoImage/blit.

**Анимация:** 
- Fly: `ag['x'] += (ag['tx'] - ag['x']) * 0.08` — экспоненциальное приближение (не кубик, но визуально норм)
- Hover: bob = sin(life * 0.05) * 3
- Glow: пульсация яркости 0.7-1.0
- Fade: alpha 1.0 → 0 за 80 кадров

### Открытая проблема

Спрайты двигаются, но при каждой итерации рендера мы обновляем каждый пиксель отдельно через coords + itemconfig. Это O(N_pixels × N_agents) операций Canvas. Для 1-3 агентов норм, для 100 — встанет.

**Решение:** PixelGrid с numpy-буфером и одним PhotoImage.

### Что поняли

Агент — это не частица, а **контейнер поведения**:
- Позиция (screen-space)
- Спрайт (пиксельная матрица)
- Состояние (idle, flying, hovering, spawning, interface)
- Дети (массив дочерних агентов)

Следующий шаг: агент может спавнить интерфейс — кнопку из пикселей.
