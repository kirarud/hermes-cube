# План: Hermes Cube → AI Avatar

## Что мы строим

Куб из частиц = физическое тело AI аватара.  
Когда AI говорит — **частицы превращаются в слова**, которые он произносит.  
Когда молчит — куб в свободной форме.

## Как это работает (схема)

```
AI ответ → парсинг текста → генерация позиций символов 
         → flip shape_cache → Morph lerp (0.8s) 
         → текст из частиц (3s) → Morph lerp обратно (0.8s)
         → возврат в исходную форму
```

Каждая частица = одна буква/символ.  
Позиция частицы = позиция буквы в текстовой раскладке (строка, колонка).  
Рендер через существующий font atlas (один draw call).

## Компоненты

### 1. `core/text_layout.py` — генератор текстовой раскладки (НОВЫЙ)

Преобразует строку текста в массив позиций (N, 3) и индексов символов.

```
get_text_positions(text: str, font_size_px: int = 12)
    → positions: NDArray[np.float64]  # (M, 3) позиции букв
    → char_indices: NDArray[np.int32]  # (M,) индексы в font atlas
    → n_used: int  # сколько частиц занято
```

**Алгоритм раскладки:**
- Разбить текст на слова (split по пробелам) + знаки препинания
- Слова выстраиваются в строки (max_width = 30 символов)
- Каждая буква = одна частица, отцентрирована в ячейке
- Пустые места (между словами, конец строки) — частицы скрываются (alive=False)
- Позиции: X = колонка × spacing, Y = строка × line_height, Z = 0

**Сложности и их решения:**
- Ширина букв разная (W ≠ I) → сейчас игнорируем, делаем monospace 1 клетка = 1 символ
- Длинный текст (> количества частиц) → обрезаем по active_count
- Короткий текст (< частиц) → лишние частицы уходят на задний план (alive=False)

```python
def update(world, dt):
    cfg = world.meta.config
    shape = cfg.get('shape_preset', 'cube')
    
    if shape == 'text':
        # shape_cache['text'] уже сгенерирован
        pass  
    else:
        # существующая логика
```

Механизм:
- `world.sim.text_shape` — словарь с `{'positions': ..., 'char_indices': ...}`
- Когда AI ответ приходит → `text_shape` заполняется через `text_layout`
- `shape_preset` переключается на `'text'`
- `morph_progress` плавно идёт от 0 до 1 (0.8 секунды)
- `char_mode` переключается на `'symbols'` (если был 'dots')
- Через 3 секунды обратный morph

### 3. Система анимации текста (В main.py или новый system)

Управляет таймингом жизни текста:

```
1. AI response приходит
2. shape_preset = 'text', morph_progress = 0
3. За 0.8s morph_progress → 1.0
4. Держим 3 секунды (текст виден, лёгкая float-анимация)
5. За 0.8s morph_progress → 0.0 (возврат к кубу)
6. char_mode обратно в 'dots' (если был symbols)
7. shape_preset обратно в предыдущую форму
```

**Float-анимация** во время демонстрации текста:
- Лёгкое синусоидальное смещение букв вверх-вниз (wave Y)
- Каждая буква слегка пульсирует по размеру (scale)
- Без rotation (текст должен быть читаем)

### 4. Совместная работа с существующими системами

| Система | Роль | Конфликт? |
|---------|------|-----------|
| **Morph** | lerp формы → текст | Используем напрямую |
| **Rotation** | вращение 3D | Когда текст — rotation OFF |
| **Animation** | wave/breathe | Когда текст — OFF |
| **Color** | depth/z_layers | Текст = все буквы одного цвета (белый/акцент) |
| **Trail** | шлейф | Работает как обычно |
| **Projection** | 2D проекция | Без изменений, scale=auto |
| **Font atlas** | рендер букв | Используем напрямую |

## Углы и риски

- **Количество частиц < символов в тексте** — обрезаем, часть текста теряется
- **Символы за границей экрана** — авто-масштабирование или перенос строк
- **Пробелы** — частица с пустым символом (alive=False), не рендерится
- **Высотой текста** можем управлять через `cube_scale` — текст на весь экран
- **Русские буквы** — font atlas их уже содержит ('rus' набор)
- **Emoji** — могут быть шире, но для простоты все символы monospace
